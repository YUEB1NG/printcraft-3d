# 3D 打印定制 · v1 规划文档（PRD-lite）

> 角色：用户 = 产品负责人（定方向）；技术联创 = 我（负责建）。
> 阶段：发现 → 规划（本文档）。后续：开发 → 打磨 → 交付。

## 1. 背景与目标

用户要做一款 **3D 打印定制产品**，首批两种定制形态：
- **透光浮雕**：上传照片（人像 / 风景 / 宠物），生成可 3D 打印的透光浮雕。
- **双面立体字**：输入文字（英文 + 数字），生成正反面不同字的双面立体字（名片座式：正面名字、背面职位）。

v1 的**唯一目标**：用一个**能发链接的网页**，验证「用户看到自己照片/文字变成可旋转的 3D 模型、并感到惊艳、想留资」这个核心假设。**不收钱、不打印、不发小程序**。

## 2. 已确认的关键决策（用户拍板）

| 决策点 | 结论 |
|---|---|
| 用途 | 正式上线接单（但 v1 只验证需求，不接单） |
| 平台 | 小程序优先；v1 先网页原型验证，小程序放 v2 |
| 定制核心 | 真·生成 3D 模型（前端实时生成） |
| 下单范围 | v1 不接支付；留资/分享 |
| 浮雕输入 | 照片（人像/风景/宠物） |
| 文字范围 | 仅英文 + 数字（避开中文字体体积坑） |
| 立体字形态 | 正反面不同字（一块板两面各刻字） |

## 3. 范围

### v1 必须有（NOW）
- 主页：两个定制入口，Apple 风格大圆角 + 材质通透感 + 按压弹簧反馈。
- 浮雕页：上传照片 → 实时生成 3D 浮雕 → 流体拖拽旋转（跟手 + 惯性）→ 参数调节（深度 / 对比度 / 尺寸 / 底光）→ 留资 / 分享。
- 立体字页：正面 + 背面文字输入 → 实时双面 3D 预览 → 材质预览 → 留资 / 分享。
- 流体交互：拖拽跟手、松手带惯性（继承速度）、按压即时反馈（Apple *Designing Fluid Interfaces* 那套）。

### v1 不做（LATER）
- 支付、真实打印履约、小程序、会员、分享裂变、AI 修图、中文字体、用户系统。

### 分阶段路线
| 阶段 | 做什么 | 交付 |
|---|---|---|
| **v1 网页体验原型** | 主页 + 浮雕页 + 立体字页（前端生成 + 惊艳预览 + 留资） | 可发链接的 demo |
| **v2 小程序 + 真下单** | 移植体验进小程序，接订单/用户/微信支付 | 能收单的小程序 |
| **v3 履约闭环** | 服务端生产级模型生成（可打印 STL、壁厚/支撑）、对接打印厂 | 完整「上传→付款→收货」 |

## 4. 技术方案（人话版）

| 能力 | 技术 | 说明 |
|---|---|---|
| 框架 | Vite + React + TypeScript | 快、轻、生态好 |
| 3D 渲染 | Three.js + @react-three/fiber + drei | 网页画 3D 最稳的组合 |
| 流体交互 | Pointer Events + 弹簧（drei/自写） | 拖拽跟手、松手惯性、按压回弹 |
| 浮雕生成 | 前端读图像素 + 顶点位移 | 照片越亮越凸、越暗越凹，在平面上顶出高低 |
| 立体字 | opentype.js + 挤出几何 | 读字体把文字变形状，挤出厚度，正反面各一组字 |
| 透光感 | 半透明 + 背光自发光材质 | 渲染看着像透光树脂（真实打印属 v3） |
| 部署 | GitHub + Vercel | 推代码一键上线，给一个链接 |

## 5. 页面结构（粗略）

- **主页 `/`**：两张大入口卡片（透光浮雕 / 双面立体字）。
- **浮雕页 `/relief`**：上传区 → 实时 3D 浮雕预览（拖拽旋转 + 惯性）→ 参数面板 → 留资 / 分享。
- **立体字页 `/text`**：正面/背面文字输入 → 实时双面 3D 预览 → 材质选择 → 留资 / 分享。

## 6. 复杂度评估

| 模块 | 难度 |
|---|---|
| 整体 | 中等偏有野心 |
| 浮雕生成 | 中等（算法简单，调好看要迭代） |
| 双面立体字 | 中等 |
| 流体交互 | 中等（有现成模式） |
| 中文字体 | v1 规避（仅英文数字） |

## 7. 验收标准（v1）

1. 主页两个入口可点进对应工具页，Pressed 状态有即时弹簧反馈。
2. 浮雕页上传一张照片后，1–2 秒内出现可旋转的 3D 浮雕，且「亮凸暗凹」符合直觉。
3. 拖拽模型时跟手；松手后按甩动速度带惯性衰减（无明显跳变）。
4. 调节深度/对比度/底光时预览实时变化。
5. 立体字页输入正反面文字后，出现正反面不同字的双面 3D 模型，可旋转预览。
6. 留资表单可提交（本地/接口占位），或生成可还原参数的分享链接。
7. 边界处理：未上传图不崩、超大图被压缩、非图片格式有提示。
8. 桌面 + 移动端均可正常操作与查看。

## 8. 用户需准备（v1 极少）

- GitHub 账号 + Vercel 账号（免费）。
- 1–2 款英文字体（如 Inter / 一款展示体）。
- 几张示例照片（用于调浮雕参数到最好看）。
